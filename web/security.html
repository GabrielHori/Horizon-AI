Security Center

Security that respects local control.

Horizon AI keeps local usage offline. Remote access is an optional layer protected by token-based authentication, rate limiting, and explicit device approval.

Local mode Offline by default
Tokens User-generated
Allowlist Configurable
Audit trail Local logs

Token-based authentication

What it does

Issues scoped tokens for remote access and per-device authorization.

Why it matters

You decide which devices can reach your local models and when.

Privacy stance

Tokens are generated and stored locally. You can revoke them instantly.

Secure remote access boundary

What it does

Creates a secure tunnel for remote access without exposing local ports.

Why it matters

Remote access stays isolated from local-only workflows.

Privacy stance

Inference still runs locally; only authorized requests traverse the tunnel.

Device management

What it does

Lists connected devices, tokens, and access history in one panel.

Why it matters

You can audit access and disable devices without downtime.

Privacy stance

Device metadata is stored locally with no third-party telemetry.

Rate limiting and IP allowlist

What it does

Enforces per-token limits and restricts access by trusted IP ranges.

Why it matters

Reduces exposure and keeps local resources protected.

Privacy stance

Rules are enforced locally with clear visibility and control.

Clear separation of concerns

Local use never depends on cloud services. Remote access is a discrete, optional layer that can be enabled or disabled at any time.

Account access
